# 5 测试

## 5.1 部署
在进行测试之前，首先需要部署整个系统环境。以下将按步骤叙述从打包、初始化到使用Docker Compose启动所有服务的过程。

### 5.1.1 环境准备与打包
确保本地环境已安装JDK 21、Node.js 20和Docker 24以上版本。
进入项目根目录，运行 `build.sh` 脚本以编译并打包后端Java应用和前端资源。

### 5.1.2 环境初始化
同样在项目根目录下，运行 `init.sh` 脚本，该脚本可能包含了设置Docker相关环境变量或其他初始化操作。

在 `~/appconfig/` 目录下的各个服务配置文件中，需要根据实际情况填充相应的配置项，例如：
- service-sms服务需要配置阿里云短信服务的endpoint、access-key-id、access-key-secret等信息。
- service-oss服务需要配置阿里云OSS的相关参数，如endpoint、access-key-id、access-key-secret以及bucket名称。
- service-order服务需配置支付宝支付网关、应用ID、私钥、公钥和站点域名等支付相关参数。

### 5.1.3 镜像构建
使用Docker Compose v1.29及以上版本，运行 `docker-compose build` 命令构建项目中所有服务的Docker镜像。

### 5.1.4 启动服务
使用 `docker-compose up -d` 命令启动所有服务容器，并以守护态模式运行。

| 通过ID获取订单     | 提供存在的订单ID时，成功从数据库获取指定订单的详细信息并返回；若订单ID不存在，返回订单不存在的错误信息。                                                 |
| 条件查询订单带分页 | 提供有效的查询条件时，返回符合条件的订单信息分页列表；若无匹配订单信息或查询条件无效，返回空分页结果。                                                   |
| 下单（生成订单）   | 当用户提供必要的下单信息并通过身份验证时，完成下单流程并成功生成新订单，返回新订单ID；若下单信息不完整或用户身份验证失败，则返回生成订单失败的错误信息。 |
| 通过ID获取订单     | 用户提供有效的身份认证并指定订单ID时，若该订单属于用户，成功返回订单详情信息；若订单ID不存在或不属于该用户，返回获取订单失败的错误信息。                 |
| 条件查询订单带分页 | 用户提供有效的查询条件并通过身份验证时，返回用户符合条件的订单列表分页结果；若无匹配订单信息或查询条件无效，返回空分页结果。                             |
| 取消订单           | 用户提供有效的身份认证并指定要取消的订单ID时，成功取消订单并返回true；若订单ID不存在或不属于该用户，返回取消订单失败的错误信息。                         |
| 获取下单支付页面   | 提供订单ID并通过用户身份验证时，成功返回指定订单ID的支付页面URL；若订单ID不存在或未通过身份验证，返回获取支付页面失败的错误信息。                        |
| 交易查询           | 根据用户ID和订单ID查询交易状态，成功返回交易状态码；若订单ID不存在或不属于该用户，返回查询失败的错误信息。                                               |
| 交易关闭           | 根据用户ID和订单ID关闭交易，成功返回操作结果状态码；若订单ID不存在或不属于该用户，返回操作失败的错误信息。                                               |
| 交易退款           | 根据用户ID和订单ID发起退款操作，成功返回退款操作结果状态码；若订单ID不存在或不属于该用户，返回操作失败的错误信息。                                       |
| 交易退款查询       | 根据用户ID和订单ID查询退款交易状态，成功返回退款交易状态码；若订单ID不存在或不属于该用户，返回查询失败的错误信息。                                       |
